# Document Patterns

## Architecture Doc

Use when describing system boundaries, data flow, server layout, deployment, and tradeoffs.

Recommended sections:

- Summary
- Problem
- Scope and non-goals
- Architecture
- Components
- Data and control flow
- Configuration
- Security considerations
- Failure modes
- Rollout plan

## ADR

Use when a decision needs a durable record.

Recommended sections:

- Title
- Status
- Context
- Decision
- Consequences
- Alternatives considered

## Runbook

Use when the reader needs to operate or troubleshoot the system.

Recommended sections:

- Purpose
- Preconditions
- Symptoms
- Investigation steps
- Mitigation
- Escalation
- Validation

## README Update

Use when onboarding or local usage matters most.

Recommended sections:

- What this project does
- Quick start
- Configuration
- Common commands
- Development workflow
- Troubleshooting
